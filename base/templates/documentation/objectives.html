Objectives

Reusability

This is priority #1. The other objectives all exist to aid in this objective. The goal is to make new projects easier and easier - not to add yet another techonology to learn

Concise and Simple

Even at the cost of run-time efficiency, it should be as simple to use as possible. This means it is more likely to be used.

Documented

If it's not documented, its existence probably won't be noticed. If it's not noticed, it won't be used. If it's not used, it's not reusable.

Consistent and Familiar

Base on consistent conventions: when possible behave like other already familiar APIs to reduce the number of new concepts that a user must learn.
